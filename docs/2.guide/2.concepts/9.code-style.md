## ESLint

对于 Nuxt，推荐的做法是通过 [`@nuxt/eslint`](https://eslint.nuxt.com/packages/module) 模块启用 ESLint 支持，该模块会为你设置基于项目的 ESLint 配置。

:::callout{icon="i-lucide-lightbulb"}
该模块针对 [新的 ESLint flat 配置格式](https://eslint.org/docs/latest/use/configure/configuration-files-new) 设计，该格式自 ESLint v9 起为 [默认格式](https://eslint.org/blog/2024/04/eslint-v9.0.0-released/)。如果你还在使用旧的 `.eslintrc` 配置，则需要 [使用 `@nuxt/eslint-config` 手动配置](https://eslint.nuxt.com/packages/config#legacy-config-format)。我们强烈建议你迁移到 flat 配置，以便具备面向未来的兼容性。
:::

## 快速设置

```bash
npx nuxt module add eslint
```

启动你的 Nuxt 应用后，项目根目录下会生成一个 `eslint.config.mjs` 文件。你可以根据需要对其进行自定义。

你可以在 [Nuxt ESLint 的文档](https://eslint.nuxt.com/packages/module) 中了解有关该模块和自定义的更多信息。
