---
title: "夜间发布通道"
description: "夜间发布通道允许使用直接从仓库最新提交构建的 Nuxt。"
---

Nuxt 每天都会合并提交、改进和 bug 修复。你可以选择加入以在下一个正式发布之前更早地测试它们。

在提交被合并到 [nuxt/nuxt](https://github.com/nuxt/nuxt) 的 `main` 分支并且**通过所有测试**之后，我们会使用 GitHub Actions 触发自动的 npm 发布。

你可以使用这些“夜间”发布来进行新功能和更改的公测。

这些“夜间”版本在构建、发布方式和质量上与稳定版本相同。唯一的区别是你需要更频繁地查看 GitHub 仓库以获取更新。审核流程和自动测试可能不会捕捉到所有回归，因此存在一些未被发现的问题的可能性。因此，我们内部会在每次正式发布前使用此通道对所有内容进行二次检查。

::note
仅在夜间发布通道可用的功能在文档中会以警示标记说明。
::

::warning
`latest` 夜间发布通道当前跟踪 Nuxt v4 分支，这意味着现在尤其可能包含破坏性变更 — 请小心！你可以通过将 `"nuxt": "npm:nuxt-nightly@3x"` 添加到依赖中来选择加入 3.x 分支的夜间发布。
::

## 选择加入

在 `package.json` 中更新 `nuxt` 依赖：

```diff [package.json]
{
  "devDependencies": {
--    "nuxt": "^4.0.0"
++    "nuxt": "npm:nuxt-nightly@latest"
  }
}
```

删除锁定文件（`package-lock.json`、`yarn.lock`、`pnpm-lock.yaml`、`bun.lock` 或 `bun.lockb`）并重新安装依赖。

## 选择退出

在 `package.json` 中更新 `nuxt` 依赖：

```diff [package.json]
{
  "devDependencies": {
--    "nuxt": "npm:nuxt-nightly@latest"
++    "nuxt": "^4.0.0"
  }
}
```

删除锁定文件（`package-lock.json`、`yarn.lock`、`pnpm-lock.yaml`、`bun.lock` 或 `bun.lockb`）并重新安装依赖。

## 使用夜间版 `@nuxt/cli`

要尝试最新版本的 [nuxt/cli](https://github.com/nuxt/cli)：

```bash [Terminal]
npx @nuxt/cli-nightly@latest [command]
```
