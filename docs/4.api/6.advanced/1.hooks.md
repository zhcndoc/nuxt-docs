## 应用钩子（运行时）

查看 [应用源代码](https://github.com/nuxt/nuxt/blob/main/packages/nuxt/src/app/nuxt.ts#L37) 以获取所有可用钩子。

| Hook                         | Arguments           | Environment     | Description                                                                                                                                                                           |
|------------------------------|---------------------|-----------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `app:created`                | `vueApp`            | Server & Client | 当创建初始 `vueApp` 实例时调用。                                                                                                                                                   |
| `app:error`                  | `err`               | Server & Client | 当发生致命错误时调用。                                                                                                                                                           |
| `app:error:cleared`          | `{ redirect? }`     | Server & Client | 当发生致命错误时调用。                                                                                                                                                           |
| `vue:setup`                  | -                   | Server & Client | 当初始化 Nuxt 根节点（root）时调用。此回调必须是同步的。                                                                                                                           |
| `vue:error`                  | `err, target, info` | Server & Client | 当 vue 错误传播到根组件时调用。[了解更多](https://vuejs.org/api/composition-api-lifecycle#onerrorcaptured)。                                              |
| `app:rendered`               | `renderContext`     | Server          | 当 SSR 渲染完成时调用。                                                                                                                                                           |
| `app:redirected`             | -                   | Server          | 在 SSR 重定向之前调用。                                                                                                                                                           |
| `app:beforeMount`            | `vueApp`            | Client          | 在挂载应用之前调用，仅在客户端侧调用。                                                                                                                                       |
| `app:mounted`                | `vueApp`            | Client          | 当 Vue 应用在浏览器中初始化并挂载完成时调用。                                                                                                                                 |
| `app:suspense:resolve`       | `appComponent`      | Client          | 当 [Suspense](https://vuejs.org/guide/built-ins/suspense#suspense) 解析事件发生时调用。                                                                                                    |
| `app:manifest:update`        | `{ id, timestamp }` | Client          | 当检测到你的应用有更新版本时调用。                                                                                                                                           |
| `app:data:refresh`           | `keys?`             | Client          | 当调用 `refreshNuxtData` 时调用。                                                                                                                                              |
| `link:prefetch`              | `to`                | Client          | 当观察到 `<NuxtLink>` 被预取（prefetched）时调用。                                                                                                                               |
| `page:start`                 | `pageComponent?`    | Client          | 当在 `NuxtPage` 内的 [Suspense](https://vuejs.org/guide/built-ins/suspense#suspense) 触发挂起（pending）事件时调用。                                                             |
| `page:finish`                | `pageComponent?`    | Client          | 当在 `NuxtPage` 内的 [Suspense](https://vuejs.org/guide/built-ins/suspense#suspense) 触发解析（resolved）事件时调用。                                                             |
| `page:loading:start`         | -                   | Client          | 当开始进行路由导航（在解析前）或当页面 key 发生变化时调用。当页面被复用时可能在页面组件的 `setup()` 不会重新运行的情况下触发（例如在 `definePageMeta` 中使用了静态 `key`）。 |
| `page:loading:end`           | -                   | Client          | 在 `page:finish` 之后调用                                                                                                                                                       |
| `page:transition:finish`     | `pageComponent?`    | Client          | 页面过渡之后调用（在 [onAfterLeave](https://vuejs.org/guide/built-ins/transition#javascript-hooks) 事件发生时）。                                                                               |
| `dev:ssr-logs`               | `logs`              | Client          | 使用传递到客户端的服务端日志数组调用（如果启用了 `features.devLogs`）。                                                                                                           |
| `page:view-transition:start` | `transition`        | Client          | 当调用 `document.startViewTransition` 之后、并且启用了[实验性的 viewTransition 支持](/docs/3.x/getting-started/transitions#view-transitions-api-experimental)时调用。 |

## Nuxt 钩子（构建时）

查看 [schema 源代码](https://github.com/nuxt/nuxt/blob/main/packages/schema/src/types/hooks.ts#L83) 以获取所有可用钩子。

| Hook                     | 参数                          | 描述                                                                                                 |
|--------------------------|------------------------------|----------------------------------------------------------------------------------------------------|
| `kit:compatibility`      | `compatibility, issues`       | 允许扩展兼容性检查。                                                                                 |
| `ready`                  | `nuxt`                       | Nuxt 初始化完成后调用，表示 Nuxt 实例已准备好开始工作。                                                 |
| `close`                  | `nuxt`                       | 当 Nuxt 实例优雅关闭时调用。                                                                         |
| `restart`                | `{ hard?: boolean }`         | 用于重启当前 Nuxt 实例。                                                                             |
| `modules:before`         | -                            | 在 Nuxt 初始化期间、安装用户模块之前调用。                                                           |
| `modules:done`           | -                            | 在 Nuxt 初始化期间、安装用户模块之后调用。                                                           |
| `app:resolve`            | `app`                        | 解析 `app` 实例后调用。                                                                             |
| `app:templates`          | `app`                        | 在生成 `NuxtApp` 期间调用，允许自定义、修改或添加构建目录的新文件（虚拟或写入 `.nuxt`）。           |
| `app:templatesGenerated` | `app`                        | 模板编译进[虚拟文件系统](/docs/guide/directory-structure/nuxt#virtual-file-system)（vfs）后调用。  |
| `build:before`           | -                            | 在 Nuxt 打包构建之前调用。                                                                             |
| `build:done`             | -                            | 在 Nuxt 打包构建完成后调用。                                                                           |
| `build:manifest`         | `manifest`                   | 在 Vite 和 webpack 构建清单时调用。允许自定义 Nitro 用于生成最终 HTML 中 `<script>` 和 `<link>` 标签的清单。   |
| `builder:generateApp`    | `options`                    | 生成应用之前调用。                                                                                   |
| `builder:watch`          | `event, path`                | 开发时文件观察器检测到项目中文件或目录变化时调用。                                                 |
| `pages:extend`           | `pages`                      | 从文件系统扫描页面路由后调用。                                                                       |
| `pages:resolved`         | `pages`                      | 页面路由携带扫描元数据增强后调用。                                                                   |
| `pages:routerOptions`    | `{ files: Array<{ path: string, optional?: boolean }> }` | 解析 `router.options` 文件时调用。后面的数组项会覆盖前面的。                               |
| `server:devHandler`      | `handler`                    | 在 Nitro 开发服务器上注册开发中间件时调用。                                                         |
| `imports:sources`        | `presets`                    | 设置时允许模块扩展导入源。                                                                           |
| `imports:extend`         | `imports`                    | 设置时允许模块扩展导入内容。                                                                         |
| `imports:context`        | `context`                    | 创建 [unimport](https://github.com/unjs/unimport) 上下文时调用。                                   |
| `imports:dirs`           | `dirs`                       | 允许扩展导入目录。                                                                                   |
| `components:dirs`        | `dirs`                       | 在 `app:resolve` 中调用，允许扩展扫描自动导入组件的目录。                                           |
| `components:extend`      | `components`                 | 允许扩展新组件。                                                                                     |
| `nitro:config`           | `nitroConfig`                | 初始化 Nitro 之前调用，允许自定义 Nitro 配置。                                                     |
| `nitro:init`             | `nitro`                      | Nitro 初始化后调用，允许注册 Nitro 钩子并直接与 Nitro 交互。                                        |
| `nitro:build:before`     | `nitro`                      | 构建 Nitro 实例之前调用。                                                                           |
| `nitro:build:public-assets` | `nitro`                   | 复制公共资源后调用，允许在 Nitro 服务器构建之前修改公共资源。                                       |
| `prerender:routes`       | `ctx`                        | 允许扩展预渲染的路由。                                                                             |
| `build:error`            | `error`                      | 构建时发生错误时调用。                                                                             |
| `prepare:types`          | `options`                    | 在 `@nuxt/cli` 写入 `.nuxt/tsconfig.json` 和 `.nuxt/nuxt.d.ts` 之前调用，允许添加自定义引用和声明到 `nuxt.d.ts`，或直接修改 `tsconfig.json` 中的选项。 |
| `listen`                 | `listenerServer, listener`   | 开发服务器启动时调用。                                                                             |
| `schema:extend`          | `schemas`                    | 允许扩展默认 schema。                                                                             |
| `schema:resolved`        | `schema`                     | 允许扩展已解析的 schema。                                                                           |
| `schema:beforeWrite`     | `schema`                     | 写入给定 schema 之前调用。                                                                         |
| `schema:written`         | -                            | 写入 schema 之后调用。                                                                             |
| `vite:extend`            | `viteBuildContext`           | 允许扩展 Vite 默认上下文。                                                                         |
| `vite:extendConfig`      | `viteInlineConfig, env`      | 允许扩展 Vite 默认配置。**在 Nuxt 5+ 中已废弃。** 在 Nuxt 5 中，操作的是共享配置，而非客户端/服务器分开配置。 |
| `vite:configResolved`    | `viteInlineConfig, env`      | 允许读取已解析的 Vite 配置。**在 Nuxt 5+ 中已废弃。** 在 Nuxt 5 中，操作的是共享配置，而非客户端/服务器分开配置。 |
| `vite:serverCreated`     | `viteServer, env`            | Vite 服务器创建时调用。                                                                           |
| `vite:compiled`          | -                            | Vite 服务器编译完成后调用。                                                                       |
| `webpack:config`         | `webpackConfigs`             | 在配置 webpack 编译器之前调用。                                                                    |
| `webpack:configResolved` | `webpackConfigs`             | 允许读取已解析的 webpack 配置。                                                                  |
| `webpack:compile`        | `options`                    | 编译前调用。                                                                                     |
| `webpack:compiled`       | `options`                    | 资源加载后调用。                                                                                 |
| `webpack:change`         | `shortPath`                  | 在 WebpackBar 的 `change` 事件时调用。                                                          |
| `webpack:error`          | -                            | WebpackBar 的 `done` 且有错误时调用。                                                           |
| `webpack:done`           | -                            | WebpackBar 的 `allDone` 事件时调用。                                                            |
| `webpack:progress`       | `statesArray`                | WebpackBar 的 `progress` 事件时调用。                                                           |

## Nitro 应用钩子（运行时，服务器端）

| `render:response`  | `response, { event }`           | 发送响应之前调用。          | [response](https://github.com/nuxt/nuxt/blob/71ef8bd3ff207fd51c2ca18d5a8c7140476780c7/packages/nuxt/src/core/runtime/nitro/renderer.ts#L24), [event](https://github.com/h3js/h3/blob/f6ceb5581043dc4d8b6eab91e9be4531e0c30f8e/src/types.ts#L38) |
| `render:html`      | `html, { event }`               | 构建 HTML 之前调用。        | [html](https://github.com/nuxt/nuxt/blob/71ef8bd3ff207fd51c2ca18d5a8c7140476780c7/packages/nuxt/src/core/runtime/nitro/renderer.ts#L15), [event](https://github.com/h3js/h3/blob/f6ceb5581043dc4d8b6eab91e9be4531e0c30f8e/src/types.ts#L38) |
| `render:island`    | `islandResponse, { event, islandContext }` | 构建 island HTML 之前调用。 | [islandResponse](https://github.com/nuxt/nuxt/blob/e50cabfed1984c341af0d0c056a325a8aec26980/packages/nuxt/src/core/runtime/nitro/renderer.ts#L28), [event](https://github.com/h3js/h3/blob/f6ceb5581043dc4d8b6eab91e9be4531e0c30f8e/src/types.ts#L38), [islandContext](https://github.com/nuxt/nuxt/blob/e50cabfed1984c341af0d0c056a325a8aec26980/packages/nuxt/src/core/runtime/nitro/renderer.ts#L38) |
| `close`            | -                              | Nitro 关闭时调用。           | -                                                                                              |
| `error`            | `error, { event? }`            | 发生错误时调用。             | [error](https://github.com/nitrojs/nitro/blob/d20ffcbd16fc4003b774445e1a01e698c2bb078a/src/types/runtime/nitro.ts#L48), [event](https://github.com/h3js/h3/blob/f6ceb5581043dc4d8b6eab91e9be4531e0c30f8e/src/types.ts#L38) |
| `request`          | `event`                        | 收到请求时调用。             | [event](https://github.com/h3js/h3/blob/f6ceb5581043dc4d8b6eab91e9be4531e0c30f8e/src/types.ts#L38)                   |
| `beforeResponse`   | `event, { body }`              | 发送响应之前调用。           | [event](https://github.com/h3js/h3/blob/f6ceb5581043dc4d8b6eab91e9be4531e0c30f8e/src/types.ts#L38), 未知                  |
| `afterResponse`    | `event, { body }`              | 发送响应之后调用。           | [event](https://github.com/h3js/h3/blob/f6ceb5581043dc4d8b6eab91e9be4531e0c30f8e/src/types.ts#L38), 未知                  |
