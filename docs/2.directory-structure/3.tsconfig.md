Nuxt [自动生成](/docs/4.x/guide/concepts/typescript#auto-generated-types) 多个 TypeScript 配置文件（`.nuxt/tsconfig.app.json`、`.nuxt/tsconfig.server.json`、`.nuxt/tsconfig.node.json` 和 `.nuxt/tsconfig.shared.json`），其中包含项目推荐的基本 TypeScript 配置，引用了[自动导入](/docs/4.x/guide/concepts/auto-imports)、[API 路由类型](/docs/4.x/guide/concepts/server-engine#typed-api-routes)、路径别名等。

您的 Nuxt 项目根目录应包含以下 `tsconfig.json` 文件：

```json [tsconfig.json]
{
  "files": [],
  "references": [
    {
      "path": "./.nuxt/tsconfig.app.json"
    },
    {
      "path": "./.nuxt/tsconfig.server.json"
    },
    {
      "path": "./.nuxt/tsconfig.shared.json"
    },
    {
      "path": "./.nuxt/tsconfig.node.json"
    }
  ]
}
```

::warning
我们不建议直接修改此文件的内容，因为这样可能会覆盖 Nuxt 或其他模块依赖的重要设置。相反，应通过 `nuxt.config.ts` 文件进行扩展。
::

::read-more{to="/docs/4.x/guide/concepts/typescript#project-references"}
在此处了解有关 Nuxt 项目不同类型上下文的更多信息。
::

## 扩展 TypeScript 配置

您可以在 `nuxt.config.ts` 文件中为每个上下文（`app`、`shared`、`node` 和 `server`）自定义 Nuxt 项目的 TypeScript 配置。
<!-- @case-police-ignore tsConfig -->
```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  typescript: {
    // 自定义 tsconfig.app.json
    tsConfig: {
      // ...
    },
    // 自定义 tsconfig.shared.json
    sharedTsConfig: {
      // ...
    },
    // 自定义 tsconfig.node.json
    nodeTsConfig: {
      // ...
    },
  },
  nitro: {
    typescript: {
      // 自定义 tsconfig.server.json
      tsConfig: {
        // ...
      },
    },
  },
})
```
