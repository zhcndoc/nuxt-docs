## 在线试玩

如果您只是想在浏览器中试玩 Nuxt 而不想搭建项目，可以使用我们的在线沙盒：

::card-group
  :card{title="在 StackBlitz 上打开" icon="i-simple-icons-stackblitz" to="https://nuxt.new/s/v4" target="_blank"}
  :card{title="在 CodeSandbox 上打开" icon="i-simple-icons-codesandbox" to="https://nuxt.new/c/v4" target="_blank"}
::

或按照下面的步骤在您的计算机上设置一个新的 Nuxt 项目。

#### 先决条件

- **Node.js** - [`20.x`](https://nodejs.org/en) 或更高（但我们推荐使用 [当前的长期支持 (LTS) 版本](https://github.com/nodejs/release#release-schedule)）
- **文本编辑器** - 没有特定的 IDE 要求，但我们推荐 [Visual Studio Code](https://code.visualstudio.com/)，并配合使用 [官方的 Vue 扩展](https://marketplace.visualstudio.com/items?itemName=Vue.volar)（此前称为 Volar）或 [WebStorm](https://www.jetbrains.com/webstorm/)，这些编辑器以及 [其他 JetBrains IDE](https://www.jetbrains.com/ides/) 都能开箱即用地提供优秀的 Nuxt 支持。
- **终端** - 用于运行 Nuxt 命令

::note
  ::details
  :summary[优化设置的附加说明：]
  - **Node.js**：请确保使用偶数版本（18、20 等）
  - **Nuxtr**：安装社区开发的 [Nuxtr 扩展](https://marketplace.visualstudio.com/items?itemName=Nuxtr.nuxtr-vscode)
  - **WSL**：如果您使用 Windows 并遇到 HMR 慢的问题，您可以尝试使用 [WSL (Windows Subsystem for Linux)](https://docs.microsoft.com/en-us/windows/wsl/install)，这可能解决一些性能问题。
  - **Windows DNS 解析缓慢** - 在 Windows 上本地开发服务器不要使用 `localhost:3000`，请使用 `127.0.0.1`，可以在浏览器中获得更快的加载体验。
  ::
::

打开一个终端（如果您使用 [Visual Studio Code](https://code.visualstudio.com)，可以打开 [集成终端](https://code.visualstudio.com/docs/editor/integrated-terminal)）并使用以下命令来创建一个新的入门项目：

::code-group{sync="pm"}

```bash [npm]
npm create nuxt@latest <project-name>
```

```bash [yarn]
yarn create nuxt@latest <project-name>
```

```bash [pnpm]
pnpm create nuxt@latest <project-name>
```

```bash [bun]
bun create nuxt@latest <project-name>
```

```bash [deno]
deno -A npm:create-nuxt@latest <project-name>
```

::

::tip
Alternatively, you can find other starters or themes by opening [nuxt.new](https://nuxt.new) and following the instructions there.
::

在 Visual Studio Code 中打开您的项目文件夹：

```bash [Terminal]
code <project-name>
```

或者在终端中切换到新项目目录：

```bash
cd <project-name>
```

## 开发服务器

现在您可以在开发模式下启动 Nuxt 应用：

::code-group{sync="pm"}

```bash [npm]
npm run dev -- -o
```

```bash [yarn]
yarn dev --open
```

```bash [pnpm]
pnpm dev -o
```

```bash [bun]
bun run dev -o

# To use the Bun runtime during development
# bun --bun run dev -o
```

```bash [deno]
deno run dev -o
```
::

::tip{icon="i-lucide-circle-check"}
做得好！浏览器窗口应该会自动为 <http://localhost:3000> 打开。
::
