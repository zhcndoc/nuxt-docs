## 发布周期

自 2023 年 1 月起，我们已采用一致的 Nuxt 发布周期，遵循 [语义化版本控制规范（semver）](https://semver.org)。我们目标为每年发布一次主框架大版本，每周发布大约一次补丁版本，每月发布大约一次小版本。除非明确标记为 `experimental`（实验性）的选项外，更新中不应包含破坏性变更。

对于 Nuxt 4 和 Nuxt 5，我们计划稍作调整。Nuxt 4 将是一个注重稳定性的版本，包含所有 `compatibilityVersion: 4` 的特性，随后不久发布的 Nuxt 5 会包含对 Nitro v3 的升级及其他变更。

此方式将破坏性变更拆分为可控阶段，以便更好地测试生态系统并实现平滑迁移。

### Nuxt 持续支持计划

我们承诺在下一个主版本发布后，至少支持前一主版本六个月，并为当前使用者提供升级路径。

### 当前版本包

当前活跃版本的 [Nuxt](https://nuxt.com) 是 **v4**，可通过 npm 使用 `latest` 标签安装 `nuxt`。

Nuxt 3 将继续获得维护更新（包括错误修复与安全补丁），维护期至 2026 年 7 月底。

每个活跃版本都有其自动生成的夜间构建版本。关于如何启用 Nuxt 夜间版本渠道，请参见 [夜间发布渠道文档](/docs/4.x/guide/going-further/nightly-release-channel)。

### 支持状态

| 状态       | 描述                                                                         |
|-------------|-------------------------------------------------------------------------------|
| 不再支持   | 该版本不再维护，不会收到安全补丁                                             |
| 维护中     | 该版本仅接收安全补丁                                                         |
| 稳定       | 该版本正在开发中并将接收安全补丁                                             |
| 开发中     | 该版本可能不稳定                                                             |
| 计划中     | 该版本尚未发布，但已在计划中                                                 |
