## "我弄不清楚如何（...）。"

你已经阅读过这些文档并且认为这是可行的，但不清楚具体方法。最好的做法是 [发起一个 GitHub 讨论](https://github.com/nuxt/nuxt/discussions)。

请不要因为问看似简单的问题而感到尴尬——我们都经历过！❤️

你遇到的每个人都是因为关心而来帮助，而不是因为有报酬。对他们最友好的做法是让他们更容易帮助你。下面是一些建议：

- _解释你的目标是什么，而不仅仅是你面临的问题。_ “我需要确保我的表单输入是可访问的，所以我正尝试让服务器和客户端之间的 id 匹配。”
- _确保你首先已阅读文档并使用你喜欢的搜索引擎。_ 让别人知道，例如写上“我在 Google 上搜索过 ‘nuxt script setup’，但找不到任何代码示例。”
- _解释你已经尝试过什么。_ 告诉别人你尝试过哪类解决方案，以及原因。通常这能让别人的建议更切合你的情况。
- _共享你的代码。_ 如果只有错误信息或截图的话，人们可能无法帮助你——但如果你以可复制粘贴的格式分享代码，情况就不同了——最好是提供一个最小可复现的示例，比如 CodeSandbox。

最后，就直接问问题吧！没有必要 [为提问而先征求许可](https://dontasktoask.com) 或者 [等别人回复你的“你好”](https://www.nohello.com)。如果这样做，你可能得不到回应，因为大家在参与前通常希望看到完整的问题。

## "可能是个 bug 吗？"

有些功能没有按文档所述工作。你不确定是否是个 bug。你已经搜索过 [open issues](https://github.com/nuxt/nuxt/issues) 和 [discussions](https://github.com/nuxt/nuxt/discussions)，但找不到相关内容。（如果存在已关闭的问题，请创建一个新的）

我们建议查看 [如何报告 bug](/docs/4.x/community/reporting-bugs)。Nuxt 仍在积极开发中，每一个问题都有助于改进它。

## "我需要专业帮助"

如果社区无法在你要求的时间内提供所需帮助，NuxtLabs 提供由 [Nuxt Experts](https://nuxt.com/enterprise/agencies) 提供的专业支持。

Nuxt Expert 的目标是在为 Vue 生态系统提供支持的同时，为那些为开源解决方案做出贡献的人创造自由职业机会，从而帮助维持生态系统的可持续性。

Nuxt 专家是被选中的 Vue、Nuxt 和 Vite 贡献者，提供专业支持和咨询服务。
