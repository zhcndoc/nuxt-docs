## 了解模块类型

**官方模块** 是以 `@nuxt/` 为前缀（作用域）的模块（例如 [`@nuxt/content`](https://content.nuxt.com)）。这些模块由 Nuxt 团队制作和积极维护。和框架一样，欢迎社区贡献，帮助他们不断改进！

**社区模块** 是以 `@nuxtjs/` 为前缀（作用域）的模块（例如 [`@nuxtjs/tailwindcss`](https://tailwindcss.nuxtjs.org)）。这些模块由社区成员制作并维护，已被广泛认可。同样，任何人都欢迎贡献。

**第三方及其他社区模块** 通常以 `nuxt-` 为前缀的模块。任何人都可以制作，用这个前缀可以让模块在 npm 上更容易被发现。这是起草和尝试新想法的最佳起点！

**私有或个人模块** 是为自己或公司使用场景制作的模块。它们不需要遵守任何命名规则来兼容 Nuxt，通常以 npm 组织作用域发布（例如 `@my-company/nuxt-auth`）。

## 列出你的模块

欢迎任何社区模块被收录在[模块列表](/modules)中。想要被收录，请在 [nuxt/modules](https://github.com/nuxt/modules/issues/new?template=module_request.yml) 仓库提交一个 issue。Nuxt 团队会帮助你在被收录前遵循最佳实践。

## 加入 nuxt-modules

将你的模块迁移到 [nuxt-modules](https://github.com/nuxt-modules) 后，会有其他人协助你，同时我们可以携手打造完美的解决方案。

如果你已有发布且运行中的模块，并希望转移到 `nuxt-modules`，请在 nuxt/modules 仓库 [提交 issue](https://github.com/nuxt/modules/issues/new)。

加入 `nuxt-modules` 后，我们可以将你的社区模块重命名为 `@nuxtjs/` 作用域下，并为其文档提供子域名（例如 `my-module.nuxtjs.org`）。
